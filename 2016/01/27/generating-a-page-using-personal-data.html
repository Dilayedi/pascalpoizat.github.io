Generating a Page using Personal Data

Jan 27, 2016

As explained in a previous post, static page generators such as Jekyll can use data stored in files to produce the information contained into HTML pages. Jekyll supports JSON, YAML, and CSV files. We will demonstrate this here by looking at how I have generated my list of academic duties.

The Data Files

The first step is to define the file(s) that will contain the information. As said before, this can be either JSON, YAML, or CSV files. Here I chose to use YAML files, but another part of my site, the list of talks, uses JSON files. Data files have to be put in the _data directory of the Jekyll tree structure. Sub-directories can be used to structure your set of data files. So here I put my data within _data/duties. Further, we will have one file for the current duties, currently.yml, and another file, before.yml, for the old ones. Here is the currently.yml one:

- name: Pedagogic Duties
  values:

  - period: since 2012
    name: Director of 1st and 2nd year of Master in CS (Applied cursus, MIAGE)
    at: Paris Ouest University

- name: Administrative Positions
  values:

  - period: since 2014
    name: Member of the Nat. Committee for Scientific Research | CoNRS Sect. 6
    at: CNRS

  - period: since 2014
    name: Member of the Laboratory Council | Conseil de Laboratoire
    at: LIP6 Laboratory

  - period: since 2013
    name: Member of the CS Advisory Council | CCD Sect. 27
    at: Paris Ouest University

You have a list of 2 items, each item containing a name (duty category) and a list of values, with a value being a period, a name (of the duty) and the location it took place (see here for information about the YAML format).

From Data to HTML

To generate HTML information for this, one can create an HTML document, duties.html, and use Liquid templating to access the data. To generate a list you simply have to type in duties.html:

---
layout: default
title: duties
---
{% for category in site.data.duties.currently %}
<h3>{{ category.name }}</h3>
<ul>
    {% for duty in category.values %}
    <li>{{ duty.period }} - {{ duty.name }}<br/>
    {{ duty.at }}
    </li>
    {% endfor %}
</ul>
{% endfor %}

As you can see, to access the data in the file _data/duties/currently.yml, we refer to site.data.duties.currently. This can be read as “the currently file that is in the duties subdirectory of the site data directory”.

For loops iterate over a list using some variable v.

{% for v in list %}...{% endfor %}

To access to some attribute a of a variable v, we use double brackets and indexing.

{{ v.a }}

The rest is self-explanatory if you know HTML.

You can get more information on Liquid templating here and here.

Factorizing Things

Now if you want to generate also the list of the old duties, you have to copy-paste the piece of code before, and replace site.data.duties.currently by site.data.duties.before. Too bad. A better way to do things is to define the code in terms of a variable duties as follows:

{% for category in duties %}
<h3>{{ category.name }}</h3>
<ul>
    {% for duty in category.values %}
    <li>{{ duty.period }} - {{ duty.name }}<br/>
    {{ duty.at }}
    </li>
    {% endfor %}
</ul>
{% endfor %}

Then you put this piece of code, say print_duties.html within the Jekyll include directory, _includes. In order to generate both list of duties you now simply have to write in duties.html:

---
layout: default
title: duties
---
<h2>Currently</h2>

{% assign duties = site.data.duties.currently %}
{% include print_duties.html %}

<h2>Before</h2>

{% assign duties = site.data.duties.before %}
{% include print_duties.html %}

If you prefer to use Markdown, use instead a duties.md file with:

---
layout: default
title: duties
---
## Currently

{% assign duties = site.data.duties.currently %}
{% include print_duties.html %}

## Before

{% assign duties = site.data.duties.before %}
{% include print_duties.html %}

Factorizing Things (revisited)

Indeed file inclusion supports parameters (see here). You can add the parameter and its value to the import as follows:

---
layout: default
title: duties
---
## Currently

{% include print_duties.html duties=site.data.duties.currently %}

## Before

{% include print_duties.html duties=site.data.duties.before %}

In the print_duties.html template that is in the _includes directory, you just have to prefix the name of the parameter with include:

{% for category in include.duties %}
<h3>{{ category.name }}</h3>
<ul>
    {% for duty in category.values %}
    <li>{{ duty.period }} - {{ duty.name }}<br/>
    {{ duty.at }}
    </li>
    {% endfor %}
</ul>
{% endfor %}
